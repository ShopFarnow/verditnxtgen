Zero-touch automation

Refreshes itself. Every single day.

You don't need to touch the server. GitHub Actions runs the full pipeline nightly at midnight IST.

⏰
Nightly cron job
A GitHub Actions workflow fires at 00:00 IST. It re-crawls every company in the database, recalculates scores, and pushes updated JSON — no server, no cost.
# .github/workflows/refresh.yml on: schedule: - cron: '0 18 * * *' # 00:00 IST jobs: refresh: runs-on: ubuntu-latest steps: - run: node scripts/crawl.js - run: node scripts/score.js - run: node scripts/generate-pages.js - uses: stefanzweifel/git-auto-commit-action
🗂️
companies.json — the live database
All scored companies live in a single JSON file, committed to GitHub. GitHub Pages serves it as an API with zero backend cost.
// companies.json (sample record) { "name": "Infosys Ltd", "cin": "L85110KA1981PLC013115", "score": 94, "verdict": "verified", "signals": { "mca": true, "domain_age_years": 26, "news_fraud": false }, "updated": "2026-05-27" }
🔍
Auto-generated SEO pages
For every company in the database, a static HTML page is generated at /company/[slug].html with full schema markup, targeting searches like "is TechNova Solutions genuine?"
// generate-pages.js (core logic) companies.forEach(co => { const html = template(co); fs.writeFileSync( `company/${slug(co.name)}.html`, html ); sitemap.push(co.name); }); // sitemap.xml re-generated each run writeSitemap(sitemap);
🤖
RAG + OpenAI validation
When a user searches an unknown company, OpenAI embeddings are used to find the closest known company, then GPT generates a plain-English verdict. Unknown company data is queued for the next nightly crawl.
// api/check.js (Cloudflare Worker) const emb = await openai.embeddings(query); const match = await pinecone.query(emb, top_k=3); const verdict = await openai.chat([ { role: "system", content: systemPrompt }, { role: "user", content: JSON.stringify(match) } ]);
